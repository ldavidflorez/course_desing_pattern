Historias de Usuario (HUs) para el Módulo de Validación

HU0: Como líder del proyecto y desarrollador, queremos planificar la creación e integración del nuevo módulo de validación, para asegurar un desarrollo organizado y alineado con objetivos.
- Criterios de Aceptación:
  - Definir alcance del módulo (funciones, patrones, integración).
  - Crear diagrama de alto nivel y plan de integración.
  - Estimar tiempo y asignar tareas en Jira.
  - Documentar plan en VALIDATION_MODULE.md.

HU1: Como desarrollador, quiero validar los datos de productos antes de crearlos, para asegurar que solo productos válidos se almacenen en la base de datos.
- Criterios de Aceptación:
  - El nombre debe ser una cadena no vacía.
  - El precio debe ser un número flotante mayor a 0.
  - La categoría debe existir en la base de datos.
  - Si los datos son inválidos, se retorna un error con mensaje descriptivo.
  - Si son válidos, el producto se crea exitosamente.

HU2: Como desarrollador, quiero validar los datos de categorías antes de crearlas, para asegurar unicidad y formato correcto.
- Criterios de Aceptación:
  - El nombre debe ser una cadena no vacía.
  - El nombre debe ser único (no existir ya en la base de datos).
  - Si los datos son inválidos, se retorna un error con mensaje descriptivo.
  - Si son válidos, la categoría se crea exitosamente.

HU3: Como desarrollador, quiero validar los datos de favoritos antes de agregarlos, para asegurar que referencien entidades existentes.
- Criterios de Aceptación:
  - El user_id debe ser un entero positivo.
  - El product_id debe ser un entero positivo y el producto debe existir.
  - Si los datos son inválidos, se retorna un error con mensaje descriptivo.
  - Si son válidos, el favorito se agrega exitosamente.

HU4: Como desarrollador, quiero integrar el módulo de validación en los servicios existentes, para centralizar y desacoplar la lógica de validación.
- Criterios de Aceptación:
  - Los servicios (ProductService, CategoryService, FavoriteService) inyectan ValidationService vía DI.
  - Antes de operaciones de creación, se llama a validate_entity con los datos.
  - Los errores de validación se propagan correctamente a los blueprints.
  - No se rompe la funcionalidad existente (endpoints siguen funcionando para datos válidos).

HU5: Como tester, quiero crear pruebas unitarias para el desarrollo actual (servicios, repositorios), para establecer una base de tests antes de añadir el módulo nuevo.
- Criterios de Aceptación:
  - Tests unitarios básicos para clases existentes (e.g., ProductService.get_all_products).
  - Tests pasan y cubren lógica esencial.
  - Se ejecutan con Pytest y generan reporte inicial de coverage.
  - Preparan el terreno para integrar tests del módulo nuevo.

HU6: Como tester, quiero tener pruebas unitarias para el módulo de validación, para asegurar que las validaciones funcionen correctamente.
- Criterios de Aceptación:
  - Tests unitarios cubren al menos 80% del código del módulo.
  - Tests incluyen casos válidos e inválidos para cada estrategia.
  - Tests pasan y se ejecutan con Pytest.
  - Reporte de coverage generado y evidencia de resultados.

HU7: Como desarrollador, quiero que el módulo siga patrones de diseño (Strategy y Chain of Responsibility), para mantener extensibilidad y claridad.
- Criterios de Aceptación:
  - Implementar IValidationStrategy y estrategias concretas.
  - Implementar ValidationHandler con cadena para validaciones secuenciales.
  - El código es legible, documentado y sigue principios SOLID.
  - Se puede extender fácilmente (e.g., añadir nueva validación sin modificar existente).

HU8: Como líder del proyecto, quiero validar la funcionalidad final del módulo e integración, para asegurar que cumple con los objetivos y no rompe el sistema existente.
- Criterios de Aceptación:
  - Ejecutar pruebas end-to-end y verificar que validaciones funcionan en endpoints.
  - Revisar que el módulo se integra sin errores en servicios y DI.
  - Confirmar que patrones de diseño se aplican correctamente.
  - Aprobar entrega final basada en criterios de aceptación de todas las HUs.
